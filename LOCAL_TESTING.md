# ローカルテスト手順

このドキュメントでは、LINE AIチャットボットのローカルテスト手順を説明します。

## 前提条件
- Python 3.10以上がインストールされていること
- 必要な依存関係がインストールされていること (`pip install -r requirements.txt`)
- `.env` ファイルが正しく設定されていること

## テスト手順

### 1. データベースの初期化
ターミナルで以下のコマンドを実行します:
```bash
python scripts/init_db.py
```

### 2. Webhookサーバーの起動
新しいターミナルで以下のコマンドを実行します:
```bash
python api/webhook.py
```

### 3. Workerサーバーの起動
別の新しいターミナルで以下のコマンドを実行します:
```bash
python api/worker.py
```

### 4. テストスクリプトの実行
さらに別のターミナルで以下のコマンドを実行します:
```bash
python scripts/test_endpoints.py
```

## 期待される動作
- テストスクリプトはWebhookとWorkerエンドポイントに対してリクエストを送信します
- 各エンドポイントから適切なレスポンスが返されることを確認します
- データベースにメッセージ履歴が保存されることを確認します
- Gemini APIが呼び出され、適切な応答が生成されることを確認します

## トラブルシューティング
- サーバーが起動しない場合: ポートの競合がないか確認してください
- データベース接続エラー: `.env` の設定を確認してください
- Gemini APIエラー: APIキーが有効か確認してください
